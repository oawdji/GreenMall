## 项目概述

GreenMall 2.0 电商平台 — 毕设/作品集项目，完整的前后端分离电商系统。

- **前端**: Nuxt 3 (Vue 3 + Vite + TypeScript + Tailwind CSS) — 目录 `client/`
- **后端**: NestJS (TypeScript + TypeORM) — 目录 `server/`
- **数据库**: MySQL 8.0
- **部署**: Docker Compose 容器化编排

## 常用命令

### Docker 开发环境

```bash
# 启动全部服务（MySQL + 后端 + 前端）
docker-compose up -d

# 停止服务
docker-compose down

# 查看日志
docker-compose logs -f server
docker-compose logs -f client

# 重建并启动（依赖变更后）
docker-compose up -d --build
```

### 前端 client/

```bash
cd client
npm install
npm run dev          # 开发模式（热更新 + SSR）
npm run build        # 生产构建
npm run preview      # 预览生产构建
npm run generate     # 静态生成（SSG）
```

### 后端 server/

```bash
cd server
npm install
npm run start:dev    # 开发模式（热重载）
npm run build        # 构建
npm run start:prod   # 生产模式
npm run seed         # 填充种子数据（测试用）
npm run migration:generate  # 生成数据库迁移
npm run migration:run       # 执行数据库迁移
```

## 技术架构

### 前端架构（Nuxt 3）

- **路由**: 基于 `pages/` 目录的文件路由，无需手动配置
- **布局**: `layouts/default.vue`（前台，含 Header/Footer）、`layouts/admin.vue`（后台，侧边栏+顶栏）
- **状态管理**: Pinia stores — `stores/auth.ts`（认证）、`stores/cart.ts`（购物车）、`stores/ui.ts`（UI状态）
- **认证**: JWT token 存储在 cookie，通过 `middleware/auth.ts` 保护路由
- **API 请求**: 通过 composables（`useApi.ts`）封装 `$fetch`，统一错误处理和 token 注入
- **SSR/SSG**: 商品列表和详情页使用 ISR（增量静态生成），其他页面 CSR

### 后端架构（NestJS）

- **模块划分**: `modules/auth`（认证）、`modules/user`（用户）、`modules/product`（商品）、`modules/category`（分类）、`modules/order`（订单+购物车）
- **认证**: JWT（`@nestjs/jwt` + `@nestjs/passport`），`jwt.strategy.ts` 实现 passport-jwt
- **数据库**: TypeORM 连接 MySQL，Entity 在各自模块的 `entities/` 目录
- **统一响应**: `TransformInterceptor` 将返回值包装为 `{ code, message, data, timestamp }` 格式
- **权限控制**: `RolesGuard` + `@Roles()` 装饰器区分 `customer` 和 `admin`
- **文件上传**: 本地磁盘存储到 `/uploads` 目录，通过 `ServeStaticModule` 提供静态访问

### 数据库核心表

users → addresses（1:N）
users → cart_items → products（N:M，cart_items 是中间表）
products ← categories（N:1）
products → product_images（1:N）
users → orders（1:N）
orders → order_items（N:1），order_items 包含商品快照（名称、图片、购买时价格）

### Docker 拓扑

```
client (Nuxt Nitro SSR, :3001) → server (NestJS, :3000) → mysql (:3306)
```

前端 Nuxt 服务端通过 `server/api/` 做 BFF 代理或客户端直连后端 API。

## 关键约定

- **价格**: 人民币 CNY，前端显示 `¥`，后端存储 `DECIMAL(10,2)` 以元为单位
- **支付**: 模拟支付 — 下单后通过 API 直接将订单状态改为 `paid`，无第三方支付集成
- **搜索**: 商品搜索使用 SQL `LIKE '%keyword%'` 模糊匹配 `name` 和 `description` 字段
- **图片**: 商品图片上传到 server 本地 `./uploads` 目录，URL 通过环境变量 `UPLOAD_URL` 拼接
- **环境变量**: 敏感配置（数据库密码、JWT secret）通过 `.env` 文件管理，`server/.env.example` 提供模板
- **订单编号**: 使用 `YYYYMMDDHHmmss` + 6位随机数生成唯一订单号
- **购物车**: 仅登录用户可用，未登录时前端引导跳转登录页

## 注意事项
